Boolean, Logical Operator, Scope

  • 실제 boolean이 아니지만 boolean처럼 평가되는 값
    
        // falsy
        Boolean(false);
        Boolean(null);
        Boolean(undefined);
        Boolean(0);
        Boolean(NaN);
        Boolean('');
        
        // truthy
        Boolean(true);
        Boolean('codeit');
        Boolean(123);
        Boolean(-123);
        Boolean({});
        Boolean([]);
  • JavaScript에서 AND와 OR연산자는 Java에서처럼 boolean값을 리턴하지 않고 왼쪽 피연산자 값의 유형에 따라서 두 피연산자중 하나를 리턴하는 형태로 동작한다.
    1. AND - 왼쪽 연산자가 falsy값일 때 왼쪽 피연산자를, truthy값일 때 오른쪽 피연산자를 리턴.
    2. OR - 왼쪽 연산자가 truth값일 때 왼쪽 피연산자를, truthy값일 때 오른쪽 피연산자를 리턴.
    
        console.log(null && undefined); // null
        console.log(0 || true); // true
        console.log('0' && NaN); // NaN
        console.log({} || 123); // {}
  • var, let, const
    1. var 키워드는 재선언, 재할당이 모두 가능.
    2. let 키워드는 재할당은 가능하지만 재선언은 불가능
    3. const키워드는 재선언, 재할당 모두 불가능.
  • 함수 스코프 (Function scope)와 블록 스코프 (block scope)
    • 함수 스코프를 가지는 변수는 해당 변수가 선언된 함수 내에서만 유효
      •     
        function sayHi() {
            var userName = 'codeit';
            console.log(`Hi ${userName}!`);
        }
        
        console.log(userName); // ReferenceError
    • 함수가 아닌 for, if, while과 같은 문법 안에서 선언한 변수는 해당 구문이 함수가 아니기 때문에 여전히 밖에서도 유효하기 때문에 중복선언과 같은 문제를 일으킨다. 이를 해결하기 위해 등장한 것이 블록스코프인데 블록스코프는 함수가 아닌 변수가 선언된 블록 내에서만 유효하다.
    • var 키워드로 선언한 변수는 함수 스코프,let과 const로 선언한 변수는 블록 스코프를 가진다.
    
        for (var i = 0; i < 5; i++) { 
            console.log(i); 
        } 
        
        console.log(i); // 5
    

    
        function sayHi() {
            const userName = 'codeit';
            console.log(`Hi ${userName}!`);
        }
        
        for (let i = 0; i < 5; i++) { 
            console.log(i); 
        } 
        { 
            let language='JavaScript' ; 
        } 
        
        console.log(userName); // ReferenceError
        console.log(i); // ReferenceError 
        console.log(language); // ReferenceError
