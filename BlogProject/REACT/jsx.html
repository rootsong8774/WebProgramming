JSX

  • JSX : JavaScript를 확장한 문법으로 XML과 비슷하게 생겼으며, 오직 React 프로젝트를 개발할 때만 사용되기 때문에 공식적인 자바스크립트 문법은 아니다.
  • 브라우저에서 실행하기 전에 바벨을 사용하여 일반 자바스크립트 형태로 코드가 변환된다.
  • XML과 비슷하기 때문에 DOM객체에서 html, xml의 문법을 거의 그대로 쓸 수 있지만, 다음에 한해서는 주의해야한다.
    • class : Javascript와 xml 모두에 class라는 키워드와 속성이 존재하기 때문에 xml에 속성으로 class를 지정할 때는 기존의 xml처럼 class가 아닌 className으로 지정해주어야한다. 물론 JSX에서 className으로 적었지만 실제 html파일에는 class로 변환된다.
    • for : xml에서 labell의 속성으로 JavaScript의 반복문 키워드로 사용되기 때문에 xml에 속성으로 for를 지정할 때는 기존의 for가 아닌 htmlFor로 지정해 주어야 한다.
    • eventHandler : 기존의 html문법에서 eventHandler는 onmouseover처럼 모두 소문자로 적었지만 JSX에서는 두 번째 단어부터 첫 글자를 대문자로 적는 camelCase로 적어야한다. (즉, onMouseOver)
  • JSX의 요소들은 반드시 단 하나의 태그로 감싸주어야 한다.
    • 아래의 경우 처럼 두 개의 p 태그를 그냥 두는 것이 아니라 div 로 감싸주어야 한다.
      •     
    ReactDOM.render(
        <div>
            <p>안녕</p>
            <p>리액트!</p>
        </div>,
        document.getElementById('root')
    );
    • 우리가 구성하고자 하는 page가 div로 감싸진 형태가 아닐 때는 다음과 같이 Fragment 태그를 써주면 된다. Fragment로 감싸고 page를 열면 html내에는 Fragment를 제외한 모든 태그가 표시된다.
      •     
    ReactDOM.render(
        <Fragment>
            <p>안녕</p>
            <p>리액트!</p>
        </Fragment>,
        document.getElementById('root')
    );
    • Fragment 태그는 다음과 같이 생략해서 사용할 수 있으므로 앞으로는 생략한 태그로 작성하도록 하자.
      •     
    ReactDOM.render(
        <>
            <h1 id="title">가위바위보</h1>
            <button className="hand">가위</button>
            <button className="hand">바위</button>
            <button className="hand">보</button>
        ,
        document.getElementById('root')
    );
